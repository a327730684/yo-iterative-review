# Claude Code 实战指南

> 一套让 AI 帮你写代码时，不跑偏、能复现、可维护的工作方法。

---

## 本指南讲什么

Claude Code 能写代码，但**怎么写**比**能不能写**更重要。

很多人用 AI 写程序时都会遇到这样的问题：

- 一开始跑得挺好，改着改着就不知道动到哪了。
- 明明改的是 A，结果 B 突然坏了。
- 想回退都找不到干净的检查点。
- 项目越大，AI 越像“闭着眼睛开车”。

本指南不讲基础操作，只讲一件事：**怎么把 Claude Code 用在真正的工程里，让它稳定、可控、可扩展地帮你写代码。**

---

## 阅读顺序

### 第一步：[Loop Engineering](./loop-engineering.md)

先建立工作方式。

Loop Engineering 是一种把复杂工程拆成多个小循环的工作思维。它不是为了慢，而是为了**每一步都走得扎实**。先读这一章，后面的所有技巧都建立在这个基础之上。

---

### 第二步：Claude Code 基础代码编写（以前端为例）

学会用 Claude Code 完成一个完整的前端功能。

这一章会以前端项目为载体，讲解：

- 如何把需求拆成 Claude Code 能理解的步骤。
- 如何写好 Prompt，让代码一次更比一次准。
- 如何在前端项目里控制文件结构、组件边界和样式规范。
- 如何利用 Claude Code 的 Agent、Review、Workflow 等能力进行高质量迭代。

读完这一章，你应该能独立用 Claude Code 完成一个中小型前端需求，而不是“让它随便写写”。

---

### 第三步：大型后端的代码控制

当前端方法放到大型后端项目里，事情会复杂得多。

这一章会讲解：

- 大型后端项目有哪些风险点（状态、事务、并发、接口兼容性等）。
- 如何用 Loop Engineering 控制大范围的代码变更。
- 如何让 Claude Code 在修改前先看清楚上下文，而不是盲人摸象。
- 如何用测试、类型、日志和代码审查守住每一轮改动。
- 多人协作时，如何把 Claude Code 的输出纳入正常研发流程。
